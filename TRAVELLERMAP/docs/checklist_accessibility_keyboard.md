# Accessibility Improvements Checklist: Keyboard Navigation & Focus

- [ ] All interactive elements (menus, filters, map controls) are accessible via keyboard (Tab, arrows, Enter)
- [ ] No action or navigation requires a mouse
- [ ] Clear visual focus indicators are present for all focusable elements
- [ ] Keyboard shortcuts are documented in help or README
- [ ] For CLI tools, all prompts and options are accessible via keyboard only
- [ ] For web/HTML outputs, tab order is logical and predictable
- [ ] No element traps keyboard focus (user can always Tab out)
- [ ] All modal dialogs or popups can be opened/closed with keyboard
- [ ] All form fields and buttons are reachable and operable by keyboard
- [ ] Accessibility features are tested with screen readers and keyboard-only navigation

_Use this checklist to guide and verify keyboard accessibility for all Traveller tools and outputs._
