# Flexible Input Parsing for Traveller Sector Tools

- [ ] Maintain a list of known sector names (e.g., Vland, Spinward Marches, etc.)
- [ ] When parsing CLI arguments, check both arguments against the sector list
- [ ] If only one argument matches a sector, assign accordingly (other is world)
- [ ] If both or neither match, prompt the user for clarification or print an error
- [ ] Add unit tests for ambiguous, valid, and invalid input cases
- [ ] Document this feature in the help text and README
