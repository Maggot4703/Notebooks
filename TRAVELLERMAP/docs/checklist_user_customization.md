# Checklist: User Customization for Traveller Tools

- [ ] Identify which features can be user-configurable (e.g., default sector, output format, base type preference)
- [ ] Design a configuration file format (e.g., JSON, YAML, INI)
- [ ] Implement config file loading and validation in scripts
- [ ] Allow command-line overrides for config options
- [ ] Document all available customization options
- [ ] Add support for saving user preferences (e.g., last used sector/world)
- [ ] Provide sensible defaults for all options
- [ ] Test with missing, partial, and invalid config files
- [ ] Ensure backward compatibility with existing CLI usage
- [ ] Update help text and README to explain customization
- [ ] Plan for future extensibility (e.g., per-user profiles, UI-based config)
