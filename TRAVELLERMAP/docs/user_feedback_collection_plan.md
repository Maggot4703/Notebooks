# User Feedback Collection Plan for Traveller Tools

## 1. Feedback Form (Recommended)
- Create a simple web-based feedback form (e.g., Google Forms, Microsoft Forms, or self-hosted HTML form)
- Include questions such as:
  - What features do you use most?
  - What is your biggest pain point?
  - What features would you like to see?
  - How satisfied are you with the current tools? (scale)
  - Open comments section
- Add a link to the feedback form in the README, CLI help text, and web UI

## 2. Email Prompt
- Provide a dedicated email address for feedback (e.g., traveller-feedback@example.com)
- Add the email address to the README, CLI help, and web UI
- Suggest users send suggestions, bug reports, or feature requests

## 3. In-App Prompt
- Add a CLI prompt (e.g., "Have feedback? Visit [feedback link] or email us!")
- Optionally, prompt for feedback after certain actions or on exit

## 4. GitHub Issues
- Encourage users to open issues for bugs or feature requests
- Add a CONTRIBUTING.md section explaining how to submit feedback via GitHub

## 5. Periodic Survey
- Send out a survey link to users (if you have a mailing list)
- Summarize results and share updates based on feedback

---

**Next Steps:**
- Choose a primary feedback method (form, email, or both)
- Create the form/email and update documentation and tool outputs
- Review feedback regularly and track actionable items
